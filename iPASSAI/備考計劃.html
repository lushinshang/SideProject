三、考古題分析重點（基於114年第4梯次試題）

📌 科目一高頻考題類型

  • ML演算法分類：哪些屬於監督式/非監督式學習（SVM、決策樹、K-means、PCA、隨機森林）
  • 特徵工程：One-hot編碼（分類變數）vs. 數值標準化（數值型）vs. 序數編碼
  • 聯邦學習（Federated Learning）：跨機構學習且資料不離開原場域
  • RAG（檢索增強生成）：索引更新、架構設計、最佳化
  • Agentic AI：搜尋策略（廣度優先/深度優先）、Solution Graph
  • MCP（Model Context Protocol）：AI Host → MCP Client → MCP Server → 資料查詢 → 回傳
  • A2A架構（Agent-to-Agent）：Client Agent發起，Remote Agent執行
  • 本地部署LLM：資料隱私優勢
  • 模型評估：MAE vs R²、交叉驗證

📌 科目二高頻考題類型

  • Prompt工程：Zero-shot、Few-shot（提供範例）、Chain-of-thought、APE（自動提示工程）、Graph Prompting
  • No-code/Low-code整合：生成式AI + No-code平台應用場景
  • RAG系統設計：增量更新索引、向量資料庫
  • AI部署策略順序：目標→資料→技術→建置（正確順序 2→3→4→1）
  • Fine-tuning vs RAG：語氣用Fine-tune，即時更新用RAG
  • 風險管理：資料最小化、去識別化、聯邦學習、同態加密
  • AI Agent類型：Agentic AI應用、Multi-agent系統
  • Context Engineering：優化提示與上下文

五、重點考試概念速查

🤖 機器學習分類

類別 代表算法
監督式學習 線性回歸、邏輯回歸、決策樹、SVM、隨機森林、KNN
非監督式學習 K-means（聚類）、PCA（降維）、關聯規則
半監督式學習 少量標籤+大量未標籤資料
強化式學習 獎勵最大化策略（如AlphaGo）

🔧 特徵工程速查

資料類型 適用方法
名義（無順序）分類變數 One-hot Encoding
有順序的分類變數 序數編碼（Ordinal Encoding）
數值型（範圍差異大） 數值標準化（Normalization / Z-score）
有時序的資料 時間序列分解

💬 Prompt工程類型

類型 說明
Zero-shot 無示範，直接問問題
Few-shot 提供少量範例後再提問
Chain-of-thought (CoT) 要求逐步推理過程
APE（自動提示工程） 自動生成與優化提示
Graph Prompting 圖結構的關係推理提示

🏗️ AI架構重要概念

  • RAG：檢索增強生成 = 向量資料庫檢索 + LLM生成；解決知識截止日期問題
  • MCP流程：AI Host → MCP Client → MCP Server → 資料查詢 → 結果回傳 AI Host
  • A2A：Client Agent發起任務 → Remote Agent執行並回傳結果
  • Federated Learning：各方不共享原始資料，僅共享模型參數/梯度
  • 本地部署LLM優勢：確保敏感資料不傳至第三方

📋 生成式AI導入步驟（考古題常考順序）

正確順序：

  1. 目標設置（定義KPI）
  2. 資料蒐集與清理
