科目一 TOP10

考點 正確答案（背誦）
⭐ AI三大技術驅動力 算力（硬體）+ 大數據 + 演算法
⭐ 弱AI vs 強AI 弱AI=單一任務（目前主流）；強AI=跨領域通用推理（未實現）
❌ ChatGPT是弱AI，不是強AI
⭐ 監督式學習的特徵 有標籤訓練資料（特徵+答案），預測新資料的輸出
⭐ 過擬合（Overfitting） 訓練集表現好，測試集表現差；解方：增加訓練資料、Dropout、正則化
⭐ 卷積神經網路（CNN） 圖像辨識首選；卷積層提取特徵，池化層降維
⭐ 負責任AI六大原則 公平、可靠/安全、隱私/安全、包容性、透明、問責
⭐ 混淆矩陣 TP/TN/FP/FN Precision=TP/(TP+FP)；Recall=TP/(TP+FN)；F1=調和平均
⭐ 遷移學習 利用預訓練模型知識用於新任務；優點：資料需求少、訓練快
⭐ 資料前處理步驟 1.蒐集→2.清洗→3.標註→4.分割（訓練/驗證/測試）→5.增強
⭐ 自然語言處理（NLP）核心任務 文本分類、情感分析、命名實體辨識（NER）、機器翻譯、摘要

科目二 TOP10

考點 正確答案（背誦）
⭐ MCP架構流程順序 AI Host → MCP Client → MCP Server → 資料查詢 → 結果回傳
❌ 不是Server→Host→Client
⭐ A2A互動流程 Client Agent發起任務 → Remote Agent執行並回傳
❌ 不是Remote主動分派
⭐ Context Engineering核心目的 優化提示與上下文
❌ 不是縮短訓練時間、不是增加參數
⭐ Agentic AI Solution Graph搜尋策略 廣度優先、深度優先或最佳優先等演算法
⭐ RAG vs Fine-tuning選擇 即時知識更新→RAG；學習特定語氣/格式→Fine-tuning
最佳組合：「能力靠Fine-tuning，知識靠RAG」
⭐ 聯邦學習用途 跨機構模型訓練，資料不離開本地，只上傳模型參數
❌ 不是同態加密
⭐ 企業導入AI第一步 明確定義目標並設定可量化KPI（非購買硬體/軟體）
⭐ 外包安全管理=何種風險策略 風險轉移（責任轉給第三方）
⭐ Chunking（文本切分）目的 提高檢索相關性與降低長上下文噪音
⭐ 台灣AI基本法 2025/12/23通過，主管機關：國科會，核心精神：以人為本，七大原則

常錯概念對比

概念A 概念B 關鍵區別
MCP（Anthropic主導） A2A（Google主導） MCP = AI↔外部工具；A2A = AI↔AI
RAG Fine-tuning RAG = 即時查知識庫（不改模型）；Fine-tuning = 修改模型權重
CoT（Chain-of-Thought） ToT（Tree of Thoughts） CoT = 線性推理步驟；ToT = 多分支方案比較
No-code（零程式碼） Low-code（少量程式碼） No-code = 非技術人員+簡單應用；Low-code = 技術人員+企業應用
Data Drift（數據漂移） Overfitting（過擬合） Data Drift = 部署後輸入分佈偏移；Overfitting = 訓練時對訓練集擬合過度
聯邦學習 同態加密 聯邦學習 = 分散訓練，資料不離開；同態加密 = 加密狀態下運算
風險轉移 風險緩解 風險轉移 = 外包/買保險，責任轉移；風險緩解 = 採措施降低風險
弱AI（Narrow AI） 強AI（General AI） 弱AI = 單一任務（現有所有AI）；強AI = 通用推理（未實現）
Precision（精確率） Recall（召回率） Precision = 預測正確中真正正確的比例；Recall = 真正正確中被找出的比例
GAN（生成對抗網路） Transformer GAN = 圖像生成主力；Transformer = GPT語言模型基礎

答題技巧

  • 情境題（40-45%）：排除法最有效，先排明顯錯誤選項，再看「最符合題目限制」
  • 注意修飾詞：例如「優先」「最主要」「核心」「不包括」→這些詞決定答案
  • 「不包括」反向題：找「最遠離核心功能」的選項
  • 順序題：萬能公式 = 目標設定→資料準備→技術選型→系統建置
  • 每題1分鐘：超過30秒記下題號，先跳過，最後回頭
